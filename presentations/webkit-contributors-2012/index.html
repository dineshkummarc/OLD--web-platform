Introducing CSSRegions

  • Create appealing digital magazines/newspapers
  • Define and arbitrarily assign flows of content between disjoint containers
  • More than one flow of content in the document -> multiple named flows
  • containers = regions -> don’t need to have the same dimensions, don’t need to have the same positioning scheme, can be styled separately

CSSRegions concepts: named flows

  • Flow: gather conceptually related content
  • Named flow: identify the flow by name

CSSRegions concepts: regions

  • Region: container that renders content from a named flow
  • Region content is replaced by named flow content
  • Not necessary a DOM element

CSSRegions concepts: region styling

  • Style the content based on the region in which the content is flown
  • Only a subset of properties can be styled

CSSRegions CSSOM

  • Expose CSSRegions concepts to scripting

CSSRegions implementation in WebKit

  • Follows last WD revision of spec
  • Runtime flag: WebKitCSSRegionsEnabled
  • Parsing of CSS properties: DONE
  • Collecting the elements into a flow: DONE
  • Display the content of flow using regions: DONE
  • Region styling: only background-color
  • CSSOM: almost DONE

Layout and rendering overview

Layout phase

  • Phase1: content is collected into named flows
  • Phase2: regions are laid out
  • Phase3: named flows are laid out

Rendering phase

  • Regions "paint" the associated part of the named flow
